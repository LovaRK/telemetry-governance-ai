```sql

  SUM(annual_license_cost) as spend
FROM agent_decisions
WHERE snapshot_id = $1
  AND tier = $2
```

## 3. RECONCILIATION CONTRACT (Hard Constraint + Audit Trail)

### The Golden Rule

**This equation MUST always be true**:

```
tier_1_spend_annual + 
tier_2_spend_annual + 
tier_3_spend_annual + 
tier_4_spend_annual
=
total_license_spend
```

**Tolerance**: ±0.01 (rounding buffer only)

### Reconciliation Logic (Correction: Hard Rejection Rule)

**Calculate**:
```typescript
const tierSum = 
  tier_1_spend_annual + 
  tier_2_spend_annual + 
  tier_3_spend_annual + 
  tier_4_spend_annual;

const delta = Math.abs(tierSum - total_license_spend);
const tolerance = 0.01;
```

### Snapshot Promotion Rule (Hard Gate)

**For executive financial metrics, reconciliation is HARD CONSTRAINT:**

```
IF delta <= 0.01:
  ✅ VALID SNAPSHOT
  - Set tier_spend_reconciled = true
  - Set tier_spend_delta = delta
  - PUBLISH to executive_kpis
  - Continue pipeline

IF delta > 0.01:
  ❌ INVALID SNAPSHOT
  - Set tier_spend_reconciled = false
  - Set tier_spend_delta = delta
  - DO NOT INSERT/UPDATE executive_kpis row
  - Log error with details: {expected: total_license_spend, actual: tierSum, delta: delta}
  - Emit governance event (reconciliation_failed)
  - Keep previous valid snapshot in production
```

**Why This Matters:**
- Tomorrow's demo credibility depends on: Total Spend = Tier1 + Tier2 + Tier3 + Tier4
- Silent mismatch would destroy customer trust immediately
- Hard rejection prevents corrupted data from reaching the dashboard
- Governance event provides audit trail

**Persist to executive_kpis** (only if valid):
- `tier_spend_reconciled` (BOOLEAN) — true only if published, false if rejected
- `tier_spend_delta` (DECIMAL) — delta value for audit

### Purpose

- **Correctness**: Numbers on dashboard are guaranteed to reconcile
- **Demo safety**: No possibility of spend numbers that don't add up
- **Auditability**: Every reconciliation attempt (success or failure) logged
- **Production-grade**: Financial metrics use hard constraints, not soft flags
- **Governance**: Reconciliation failures trigger investigation
- **P0.5 support**: Failed reconciliations appear in governance ledger

## 5. SNAPSHOT PROMOTION RULE (Hard Gate for Demo Safety)

### The Golden Rule

**For executive financial metrics: reconciliation is HARD CONSTRAINT, not soft flag**

Rationale: Customer demo tomorrow depends on:
```
Total Annual License Spend 
= 
Tier 1 Spend + Tier 2 Spend + Tier 3 Spend + Tier 4 Spend
```

If these numbers disagree, dashboard loses all credibility immediately.

### Rule: Reject Invalid Snapshots

```
Reconciliation Delta (absolute difference between tier sum and total spend)

VALID:   delta ≤ 0.01
  ↓
  ✅ Accept snapshot
  → Persist to executive_kpis table
  → Publish to API
  → Display on dashboard
  → Update governance ledger with SUCCESS

INVALID: delta > 0.01
  ↓
  ❌ Reject snapshot
  → DO NOT insert/update executive_kpis
  → Keep previous valid snapshot in production
  → Log error with {expected, actual, delta} details
  → Emit governance event: reconciliation_failed
  → Alert operator in logs
  → Display warning in admin panel (not customer-facing)
```

### Implementation

**In aggregation-service.ts**, before calling upsertExecutiveKpis():

```typescript
const tierSum = tier_1_spend + tier_2_spend + tier_3_spend + tier_4_spend;
const delta = Math.abs(tierSum - total_license_spend);
const tolerance = 0.01;

if (delta > tolerance) {
  // REJECT: Invalid snapshot
  logger.error('Tier spend reconciliation FAILED', {
    expected_total: total_license_spend,
    actual_tier_sum: tierSum,
    delta: delta,
    tolerance: tolerance,
    snapshot_id: snapshotId,
    pipeline_run_id: pipelineRunId
  });
  
  // Emit governance event
  await governance.emit('reconciliation_failed', {
    type: 'tier_spend_mismatch',
    severity: 'error',
    snapshot_id: snapshotId,
    delta: delta
  });
  
  // DO NOT persist to database
  // Previous valid snapshot remains in production
  return {
    success: false,
    reason: 'Tier spend reconciliation failed',
    keep_previous_snapshot: true
  };
}

// ACCEPT: Valid snapshot
await upsertExecutiveKpis({
  ...kpiData,
  tier_spend_reconciled: true,
  tier_spend_delta: delta
});
```

### Audit Trail

Every reconciliation attempt (success or failure) is logged:
- **Success**: snapshot_id, reconciled=true, delta value, published_at
- **Failure**: snapshot_id, reconciled=false, delta value, reason='delta > 0.01', rejected_at

This allows P0.5+ to surface reconciliation health in governance dashboard.
