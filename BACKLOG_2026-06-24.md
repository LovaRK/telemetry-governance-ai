#### Deliverables

| File | Purpose |
|---|---|
| `install.sh` | Mac/Linux installer (Bash). Full flow below. |
| `install.ps1` | Windows installer (PowerShell). Same flow, winget/UAC instead of brew/sudo. |
| `doctor.sh` | Mac/Linux health-check. Single command that diagnoses: ✅/❌ on Git, Docker daemon, Ollama, Postgres container, Web dashboard, Worker, Splunk connection, Splunk token validity, .env secrets present. Massive support-time saver. |
| `doctor.ps1` | Windows health-check. Same checks via PowerShell + curl + docker exec. |
| `common.env.template` | Single source of truth for the `.env` schema, with `<RANDOM_HEX_64>` markers the install scripts replace with `openssl rand -hex 32`. Avoids drift between the two installers. |
| `README.md` | The one-liner usage notes + a 30-second "what's about to happen" explainer + troubleshooting → doctor recipe. |

#### Install flow (both scripts execute the same logical sequence)

```
Check OS                       → branch to platform-specific package mgr
Check admin rights             → fail-fast with a clear "rerun with sudo / elevated PowerShell" message
Check disk space               → require ≥ 20 GB free (5 GB Ollama + 5 GB Docker images + 10 GB room)
Check RAM                      → branch into 3 LLM options based on detected RAM
Install Git                    → brew install git (Mac) / winget install Git.Git (Windows) / apt-get (Linux)
Install Docker Desktop         → brew install --cask docker / winget install Docker.DockerDesktop
Wait for Docker daemon         → loop docker info until healthy with a "please launch Docker Desktop" prompt
Install Ollama (if chosen)     → brew install ollama / winget install Ollama.Ollama
Pull LLM model                 → ollama pull <chosen-model> with progress dots
Clone or update repo           → if not present, git clone; if present, git pull
Generate .env                  → read common.env.template, replace markers with openssl rand -hex 32, prompt admin creds
docker compose up -d           → with progress + a "this can take 2-3 min on first build" notice
Health checks                  → loop until postgres/web/worker all healthy
Open dashboard                 → print http://localhost:3002, admin email/password, "click Refresh to run first pipeline"
```

#### RAM-based LLM model selection (`install.sh` and `install.ps1`)

Before downloading any model the script detects RAM and shows a menu:

```
Detected RAM: 8 GB

Choose your LLM provider:
  1. llama3.2:3b      — 2 GB download, ~12 GB RAM during inference (recommended for your machine)
  2. gemma2:9b        — 5 GB download, ~20 GB RAM during inference (NOT recommended — may swap)
  3. Anthropic API    — no download, ~50 ms latency, requires ANTHROPIC_API_KEY

Press 1 / 2 / 3, or Enter for the recommended option:
```

Same machine with 16+ GB RAM defaults `2. gemma2:9b` recommendation (matches policy). 32+ GB always recommends 2. This avoids the "installer hung downloading 5 GB on a low-spec laptop" failure mode.

#### Distribution model — explicit decision

- ✅ **For Tejas + internal use**: short `curl ... | bash` and `iwr ... | iex` one-liners. Documented in README.md as the internal fast path.
- ❌ **For enterprise customers**: NOT `curl ... | bash` (many enterprises block piping remote scripts to shell). Instead:
  1. `git clone https://github.com/LovaRK/telemetry-governance-ai && cd telemetry-governance-ai`
  2. `./scripts/install/install.sh` (or `.\scripts\install\install.ps1`)
- The two paths run the same scripts; only the discovery / trust mechanism differs.

#### Long-term roadmap (NOT this PR — captured for the future)

Native double-click installers — pure ergonomics + per-user experience:
- `Telemetry-Agent-Installer.pkg` (Mac) — bundles all scripts + Docker Desktop license terms + uninstall registration
- `Telemetry-Agent-Installer.exe` (Windows) — same, with Start menu shortcut + Add/Remove Programs entry

Benefits:
- Double-click install (no terminal needed)
- Desktop shortcut to dashboard
- Uninstall via system tooling
- Auto-update via signed manifests
- Far better non-technical client experience

Skip for now; deliver the shell installer first since it's a 1-week task vs. 4–6 weeks for signed native installers.
