**Critical Observation:** Every single index, regardless of event count (174K vs 0), is marked ELIMINATE.

### Root Cause: Hardcoded Fallback Behavior

**Code Location:** `/apps/api/agents/llm-decision-agent.ts:340-428` in `buildPartialResponseFromPrecomputedScores()`

When LLM is unavailable, the system automatically assigns actions based on composite score:

```typescript
function buildPartialResponseFromPrecomputedScores(inputs: RawTelemetryInput[], config: UserConfig) {
  const decisions: LLMDecision[] = inputs.map((input) => {
    const scores = input.precomputedScores || {
      utilizationScore: 0,
      detectionScore: 0,
      qualityScore: 100,
      compositeScore: 0,      // ← DEFAULT = 0
      tier: 'Low-Value',
      detectionGap: false,
      operationalGap: false,
    };

    // Deterministic fallback logic
    let action: 'KEEP' | 'OPTIMIZE' | 'ARCHIVE' | 'ELIMINATE' | 'S3_CANDIDATE' = 'KEEP';
    if (scores.compositeScore >= 65) {
      action = 'KEEP';
    } else if (scores.compositeScore >= 40) {
      action = 'KEEP';
    } else if (scores.compositeScore >= 20) {
      action = 'OPTIMIZE';
    } else {
      action = 'ELIMINATE';  // ← DEFAULT WHEN compositeScore < 20
    }
    
    // Hardcoded confidence for fallback
    confidence: 'LOW' as const,
    confidenceScore: 0.5,     // ← ALWAYS 50%, NOT 90%
  });
}
```

### Why This Happens

1. **LLM Unavailable:** If `gemma2:9b` Ollama instance is down or Anthropic API not configured
2. **Fallback Triggered:** System uses `buildPartialResponseFromPrecomputedScores()`
3. **Default Scores:** If precomputedScores not injected, composite_score defaults to **0**
4. **compositeScore < 20 → ELIMINATE** (hard rule)
5. **Confidence = 0.5** (50%, not 90%)
